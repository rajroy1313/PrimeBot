Adding PrimeBot to your server is simple:

  1. Click the invite link: Invite PrimeBot
  2. Select your Discord server from the dropdown
  3. Make sure you have "Manage Server" permissions
  4. Review and approve the permissions
  5. Click "Authorize" to complete the process
Note: You need to have "Manage Server" permissions to add bots to a Discord server.

PrimeBot requires these permissions to function properly:

  • Send Messages - To respond to commands and send notifications
  • Embed Links - To send rich, formatted messages
  • Add Reactions - For giveaways and polls
  • Attach Files - To send images and files when needed
  • Manage Messages - To clean up and manage content
  • Manage Channels - For the ticket system
  • Manage Roles - For leveling system rewards
Tip: Granting Administrator permissions makes setup easier, but it's not required for basic functionality.

After adding PrimeBot, follow these steps:

  1. Use /help to see all available commands
  2. Configure welcome messages with /welcomeconfig
  3. Set up the leveling system with /leveling setup
  4. Configure the ticket system with /ticket setup
  5. Test basic functionality with /about or /stats
Quick setup commands:
/help
/about
/stats

PrimeBot supports both command types:

  • Slash Commands (/): Modern Discord commands that auto-complete and show help text. Recommended for new users.
  • Prefix Commands ($): Traditional commands that start with a prefix. Still supported for compatibility.
Examples:
/help (slash command)
$help (prefix command)
Recommendation: Use slash commands for the best experience with auto-completion and parameter hints.

There are several ways to see available commands:

  • Use /help for a complete command list
  • Type / and look for PrimeBot commands in the auto-complete
  • Visit our documentation page for detailed explanations
  • Use /categories to browse commands by category
/help - Complete command list
/categories - Browse by category
/about - Bot information

Creating a giveaway is easy with the /giveaway command:

/giveaway duration:1d winners:1 prize:Discord Nitro

Parameters:

  • duration: How long the giveaway runs (s/m/h/d)
  • winners: Number of winners to select
  • prize: Description of what you're giving away
Duration examples: 30s, 15m, 2h, 7d

Yes! You can manage giveaways with these commands:

/end message_id:123456789 - End a giveaway early
/reroll message_id:123456789 - Pick new winners

To get the message ID:

  1. Right-click on the giveaway message
  2. Select "Copy Message ID" (Developer Mode must be enabled)
  3. Use the ID in your command
Tip: Enable Developer Mode in Discord Settings > Advanced > Developer Mode to see message IDs.

The leveling system tracks member activity and awards XP:

  • XP per message: 15-25 XP (random amount)
  • Cooldown: 60 seconds between XP gains
  • Level formula: Level × 100 XP required to level up
  • Rewards: Roles can be assigned at certain levels
/leveling setup - Configure the system
/stats - Check your level and XP
Note: The cooldown prevents spam and ensures fair XP distribution.

Set up automatic role rewards for reaching certain levels:

  1. Create roles in your server settings
  2. Use /leveling roles to configure level rewards
  3. Set which role is awarded at which level
  4. Members automatically receive roles when leveling up
/leveling roles - Configure level roles
/leveling channel - Set announcement channel
Tip: Make sure PrimeBot has "Manage Roles" permission and its role is above the reward roles.

Setting up tickets involves a few configuration steps:

  1. Create a category for ticket channels
  2. Use /ticket setup to configure the system
  3. Set the ticket category with /ticket category
  4. Configure support roles with /ticket role
  5. Test by creating a ticket with /createticket
/ticket setup - Initial configuration
/ticket category - Set ticket category
/ticket role - Configure support roles

Members can create tickets for private support:

/createticket subject:"Need help" description:"Describe your issue here"

What happens when a ticket is created:

  • A private channel is created in the ticket category
  • Only the member and support roles can see it
  • Support staff are notified of the new ticket
  • The ticket can be closed when resolved
Features: Automatic logging, ticket history, and archive functionality.

Try these troubleshooting steps:

  1. Check bot status: Is PrimeBot online in your member list?
  2. Verify permissions: Does the bot have "Send Messages" in the channel?
  3. Try slash commands: Use /help instead of $help
  4. Check channel permissions: Make sure the bot can read and send messages
  5. Re-invite if needed: Use the invite link to refresh permissions
Quick test: Try /about in a channel where you know the bot has permissions.

Common leveling issues and solutions:

  • Not gaining XP: Check the 60-second cooldown between messages
  • No level announcements: Set an announcement channel with /leveling channel
  • Roles not being assigned: Ensure bot has "Manage Roles" permission
  • XP not showing: Use /stats to check your current level and XP
  • System disabled: Use /leveling setup to enable it
/leveling setup - Enable/configure leveling
/stats - Check your level
/leveling channel - Set announcement channel

Common giveaway issues:

  • No reaction button: Bot needs "Add Reactions" permission
  • Can't end giveaway: Check message ID format and use /end
  • Embed not showing: Bot needs "Embed Links" permission
  • Winner selection failed: Ensure participants joined before the giveaway ended
/giveaway duration:1m winners:1 prize:Test - Quick test giveaway
/end message_id:YOUR_MESSAGE_ID - End early for testing
Test tip: Create a 1-minute test giveaway to verify everything works.
